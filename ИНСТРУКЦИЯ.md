# 🤖 ПП Рецепты — Telegram Bot
## Инструкция по запуску

---

## Шаг 1 — Узнай свой числовой Telegram ID

1. Напиши боту @userinfobot в Telegram
2. Он пришлёт твой ID — например: `123456789`
3. Запомни это число — это твой `ADMIN_ID`

---

## Шаг 2 — Добавь бота администратором в канал

1. Открой канал "ПП Рецепты" в Telegram
2. Настройки → Администраторы → Добавить администратора
3. Найди своего бота по username (который ты создал в BotFather)
4. Дай право: **"Пригласить пользователей"** ✅
5. Сохрани

---

## Шаг 3 — Загрузи код на GitHub

1. Зайди на github.com → создай аккаунт если нет
2. Нажми "New repository" → назови `pp-bot` → Create
3. Загрузи все файлы из этой папки в репозиторий
   (можно перетащить мышкой прямо на сайте)

---

## Шаг 4 — Запусти на Render.com (бесплатно)

1. Зайди на render.com → Sign up (через GitHub)
2. New → **Background Worker**
3. Подключи свой репозиторий `pp-bot`
4. Настройки:
   - **Name**: pp-recipes-bot
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python bot.py`

5. Прокрути вниз до **Environment Variables** и добавь:

| Key | Value |
|-----|-------|
| BOT_TOKEN | `8889521233:AAE25F26rz3gc_SnBXKdn9aekTi4rcU6sMU` |
| ADMIN_ID | `твой числовой ID из шага 1` |
| CHANNEL_ID | `-1001003358660639` |
| KASPI_LINK | `https://pay.kaspi.kz/pay/tmchkblz` |

6. Нажми **Create Background Worker**
7. Подожди 2-3 минуты пока деплоится

---

## Шаг 5 — Проверь что всё работает

1. Напиши своему боту `/start`
2. Должно появиться приветственное сообщение с кнопками
3. Нажми "Купить" — должна появиться Kaspi ссылка
4. Отправь боту любой скриншот — тебе должно прийти уведомление
5. Нажми "Одобрить" — бот должен отправить invite-ссылку

---

## Команды для тебя (администратора)

| Команда | Что делает |
|---------|-----------|
| `/users` | Список всех подписчиков |
| `/stats` | Статистика: кол-во мүшелер и доход |

---

## ⚠️ Важно

- Не публикуй BOT_TOKEN нигде публично
- Render бесплатный план: бот работает стабильно для Background Worker
- База данных (subscribers.db) хранится на сервере Render
